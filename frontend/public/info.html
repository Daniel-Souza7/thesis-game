How to Play

Objective:

Decide when to exercise your option to maximize your payoff. You compete against a trained AI algorithm. Can you beat the machine?

Game Flow:

  • Hold: Wait and bet that the next price movement is favorable
  • Exercise: Lock in your current payoff and close your game

Important Notes:

  • Machine Decision: The AI's decision is revealed after yours
  • Barriers: If a barrier is hit, your payoff becomes $0
  • Payoff Formula: Each game has a unique payoff structure shown in the info panel

Difficulty Levels:

  • Medium: Simple barrier options with the simplest payoffs
  • Hard: Moving barrier or more complex payoffs
  • Impossible: Exotic options with multiple features

Game Payoffs

Medium Difficulty:

UpAndOut Call: Pays (S(t) - K)⁺ if stock never exceeds upper barrier. Single stock, simple up barrier, simple call.

DownAndOut MinPut: Pays (K - min(S₁,S₂,S₃))⁺ if no stock drops below barrier. 3 stocks, put on current minimum across all stocks.

DoubleBarrier MaxCall: Pays (max(S₁...S₇) - K)⁺ if all stocks stay within barriers. 7 stocks, call on current maximum.

Hard Difficulty:

RandomlyMovingBarrier Call: Upper barrier moves uniformly randomly (-2;1) in each step. Must avoid dynamic barrier.

UpAndOut MinPut: Pays (K - min(stocks))⁺ with upper barrier. Put payoff on current weakest stock, 3 stocks.

DownAndOut Best2Call: Pays on average of top 2 stocks if no stock hits lower barrier. 7 stocks, selective averaging.

Impossible Difficulty:

DoubleBarrier Lookback Put: Pays (max price so far - current price)⁺. Path-dependent with double barriers.

DoubleBarrier RankWeighted Call: Ranks 3 stocks, applies weights (15%, 50%, 35%) to 1st, 2nd, 3rd. Complex double barriers basket.

DoubleMovingBarrier Dispersion: Pays max(std_dev(stocks) - 1, 0). Both barriers move randomly. Measures volatility spread.

The Underlying Process

Rough Heston Model:

All stock paths are generated using the Rough Heston model, a more realistic alternative to classical Brownian motion. Research shows that volatility exhibits rough behavior (H ≈ 0.1) rather than smooth Brownian paths (H = 0.5).

Model Calibration:

Based on "SPX Calibration of Option Approximations under Rough Heston Model" by Siow Woon Jeng & Adem Kiliçman (Mathematics, 2021), we calibrated the model to real SPX options data.

Stochastic Differential Equations:

dSₜ = μ Sₜ dt + √Vₜ Sₜ dWₜ
Vₜ = V₀ + 1/Γ(H+½) ∫₀ᵗ (t-s)^(H-½) [λ(θ - Vₛ)ds + ν√Vₛ dZₛ]

where dWₜ and dZₜ are correlated Brownian motions with correlation ρ

Calibrated Parameters:

• Hurst exponent: H = 0.03 (very rough)
• Initial variance: V₀ = 0.026
• Long-term variance: θ = 0.07
• Mean reversion speed: λ = 0.5
• Vol-of-vol: ν = 0.29
• Correlation: ρ = -0.75 (leverage effect)
• Drift: μ = 0.02

Why Rough Heston?

The low Hurst parameter (H = 0.03 << 0.5) captures the jagged, non-smooth behavior observed in real market volatility, making it more challenging and realistic for optimal stopping decisions.

About the Machine

The Algorithm: RT

The machine uses RT (Enhanced Randomized Neural Networks Algorithm), a universal and powerful algorithm for optimal stopping problems that handles both path-dependent and non-path-dependent payoffs.

How It Works:

1. Backward Induction: Works backward from maturity (t=12) to present (t=0)
2. At each time step:
  • Evaluates immediate exercise value (uses full path history for barriers)
  • Estimates continuation value (expected future payoff if holding)
  • Decides: Exercise if immediate > continuation, else Hold
3. Learning Continuation Value:
  • Uses randomized neural network with GELU activation
  • Fits coefficients via least squares regression
  • Includes current payoff as input feature for better learning
  • Trains only on in-the-money paths for efficiency

Training Parameters:

• Training paths: 15,000,000 simulated paths
• Test paths: 2,000 paths
• Hidden neurons: 40
• Activation: GELU
• Time steps: 12 (monthly over 1 year)
• Train ITM only: Yes
• Use payoff as input: Yes

Important Note:

The machine has NOT been trained on the real game paths you play!
It was trained on separate simulated paths from the same Rough Heston model. When you play, both you and the machine see the path for the first time, but the machine applies its pre-learned strategy.
